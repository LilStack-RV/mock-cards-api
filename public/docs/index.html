UNC Case Competition Cards API

We recommend using the postman application for testing api endpoints, which can be downloaded here.

Routes:

GET /api/v1/cards

POST /api/v1/cards/:id/apply


Details:

GET /api/v1/cards

  • Takes in no values in the request body, or as request parameters
  • Returns a json object with all 4 available credit cards and their associated data, such as the name of the card, the company that provides the card, credit score ranges where approval is high, and APR data.

POST /api/v1/cards/:id/apply

  • Takes in the numerical 'id' of the card being applied for as a query string parameter
  • Takes in the credit score of the applicant in the JSON body of the request. ex: { creditScore: 550 }
  • Expects an incoming header with key: 'Content-Type' and value: 'application/json'

Examples:

POST /api/v1/cards/82457245/apply

  • Returns a json object with success or failure of the application, as well as a message detailing the reason for success or failure
